# 存储（WebDAV）

Bastion 支持 WebDAV 作为远端备份目标。

WebDAV 凭据会以加密形式存储在 Hub 数据库中，并且 **按节点隔离**：

- Hub 的节点 ID 固定为 `hub`
- 每个客户端都有独立的凭据命名空间

这意味着：

- 在客户端上运行的任务只能使用该客户端已有的 WebDAV 凭据。
- 仅在 Hub 上创建凭据并不足以让客户端运行的任务使用它。

## 管理 WebDAV 凭据

在 Web UI：**设置 → 存储**（默认 Hub 节点）

存储页面按节点上下文展示（`hub` 或某个客户端）。你可以：

- 按名称创建/编辑/删除凭据
- 查看更新时间
- 复制凭据名称用于任务配置

## 分发 WebDAV 凭据到客户端

把 Hub 的某个凭据复制到多个客户端：

1. 确保该凭据已存在于 **Hub**（节点 `hub`）
2. 点击 **分发到客户端**
3. 选择目标客户端（按标签筛选或手动填写客户端 ID）
4. （可选）勾选 **覆盖**
5. 先预览，再创建批量操作

进度与结果可在 **设置 → 批量操作** 中查看。

## WebDAV 凭据的使用场景

- **备份目标（WebDAV）**：创建任务时选择 WebDAV 目标，需要选择一个凭据名称。
- **恢复到 WebDAV**：恢复使用“执行节点”的 WebDAV 凭据命名空间；如果你把恢复放到客户端上执行，可能需要先将同名凭据分发到该客户端。

另见：[恢复与校验](/zh/user/restore-verify)。
