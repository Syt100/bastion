# 备份快照（Backup snapshots）

**快照（Snapshot）** 是一次成功运行产生的产物：包括存储在备份目标中的实际备份数据，以及 Hub 中的一条索引记录。

在 Web UI 中，快照是一等资源：你可以 **固定**、**删除**，并配置/执行 **保留策略**。

## 在哪里查看快照

在 Web UI：

- **备份任务** → 进入某个任务 → **快照**

快照按任务列出，通常包含运行时间、目标、格式、大小/统计信息（若可用）以及状态。

## 状态含义

- `present`（可用）：快照存在且可用
- `deleting`（删除中）：删除任务已入队或正在执行
- `deleted`（已删除）：快照已成功删除
- `missing`（缺失）：在备份目标中找不到快照（视为“已经不在了”）
- `error`（错误）：删除失败，可能正在重试/被阻塞/已放弃（查看 **删除日志**）

## 固定（Pin）快照

固定是一种安全机制：

- 被固定的快照不会被自动保留策略删除。
- 手动删除固定快照需要在 UI 中进行更明确的 **强制删除** 确认。

## 删除快照（单个 / 批量）

删除是异步的（避免 HTTP 超时，并支持重试）。

典型流程：

1. 选择一个或多个快照
2. 点击 **删除** 并确认
3. 快照会进入 `deleting`（删除中）状态，后台任务开始执行
4. 通过 **删除日志** 查看进度、错误，以及重试/忽略操作

若删除失败：

- **立即重试**：立即重新入队
- **忽略**：将该删除任务标记为忽略（例如你已经在 Bastion 外部手动删除了数据）

## 保留策略

保留策略由服务端强制执行，用于控制存储占用。

在任务编辑器中可配置：

- **保留最近 N 份**：保留最近 N 个快照
- **保留最近 N 天**：保留最近 N 天内的快照
- **删除上限（每次 / 每天）**：安全阀，避免一次性删除过多

UI 支持：

- **预览**：预览将要删除的快照
- **立即执行**：立即入队执行保留策略删除

已固定的快照永远不会被保留策略自动删除。

## 多节点说明（Hub vs 客户端）

- `local_dir` 目标：快照数据实际保存在执行该任务的节点上（Hub 或某个客户端）。删除任务必须在该节点上执行，因此离线客户端会导致删除延后。
- `webdav` 目标：快照保存在远端 WebDAV 存储中。删除需要执行节点具备对应 WebDAV 凭据（通常是 Hub，除非你已将凭据分发到客户端）。
