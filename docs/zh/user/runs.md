# 运行记录（Runs）

运行记录（Run）是一次任务执行，会记录状态、时间、进度以及事件日志。

## 在哪里查看

在 Web UI：

- **备份任务** → 选择一个任务 → **运行记录**
- 点击某条运行记录进入详情页

## 状态含义

- **queued**：已进入队列，等待执行
- **running**：正在所选节点上执行
- **success**：执行成功（可能产出快照）
- **failed**：执行失败
- **rejected**：因重叠策略被拒绝（例如 overlap=reject 且已有运行正在进行）

## 运行详情（可以做什么）

运行详情页包含：

- **摘要**：状态、时间、基础指标，以及本次使用的 source/target
- **运行事件**：运行期间的增量事件/进度（WebSocket）
- **操作**：从该运行发起的恢复/校验

对于成功的运行，你还可以发起：

- **恢复（Restore）**：恢复到指定目的地
- **校验（Verify）**：恢复到临时目录并校验文件哈希

另见：[恢复与校验](/zh/user/restore-verify)。

## 运行记录保留（Run retention）

Hub 会按运行时配置中的 **Run retention days** 自动清理旧的运行记录。

重要行为：

- 运行记录清理与快照关联：只要该成功运行仍然有“存活”的快照（present/deleting/error），对应的运行记录会被保留。
- 当快照被删除（或被保留策略完全删除）后，运行记录会在超过保留截止时间后被清理。

另见：[运行时配置](/zh/user/operations/runtime-config)。
